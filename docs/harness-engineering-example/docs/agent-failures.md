# Agent Failures — 에이전트 실패 로그

에이전트가 범한 실수와 그 구조적 대응을 기록합니다.
이 문서는 하네스 진화의 시작점입니다.

## 승격 정책

- 같은 패턴 **2회 반복** → `golden-principles.md`에 원칙 추가
- 같은 패턴 **3회 반복** → 린트/테스트로 **구조적 불가능화** 필수
- 월 1회 GC 에이전트가 전체 검토

## 실패 기록

| ID | 날짜 | 증상 | 근본 원인 | 강화책 | 도메인 |
|----|------|------|-----------|--------|--------|
| F-0001 | (예시) | 워크트리 없이 develop에서 직접 코드 수정 | CLAUDE.md의 워크트리 규칙을 무시 | pre-edit hook에서 차단 (exit 2), core-beliefs #1 | 워크플로우 |
| F-0002 | (예시) | .env 파일이 커밋에 포함됨 | git add -f 사용, .gitignore 우회 | 04-security 검증기에서 git ls-files 체크 추가 | 보안 |

> 위 예시를 삭제하고 실제 실패를 기록하세요.

## 기록 방법

새 실패 발생 시 아래 항목을 채워 행을 추가합니다:

- **ID**: `F-YYYY-MM-DD-N` (같은 날 여러 건이면 N 증가)
- **증상**: 외부에서 관찰 가능한 현상 (에러 메시지, 잘못된 동작)
- **근본 원인**: "try harder"로 해결 안 되는 구조적 사유
- **강화책**: ADR, 린트 규칙, 검증기, 문서 링크 등 구체적 대응
- **도메인**: 워크플로우 / 보안 / 아키텍처 / 테스트 / 문서 등
