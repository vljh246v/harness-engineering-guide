# 코드 리뷰 스킬

이 스킬은 PR 리뷰 또는 코드 리뷰 요청 시 사용합니다.

## 리뷰 체크리스트

### 1. 아키텍처 적합성
- [ ] 레이어 의존성 방향이 ARCHITECTURE.md와 일치하는가?
- [ ] 새 클래스/모듈이 적절한 레이어에 위치하는가?
- [ ] 경계를 넘는 불필요한 의존성이 없는가?

### 2. 코드 품질
- [ ] 함수/메서드가 단일 책임을 갖는가?
- [ ] 중복 코드가 없는가? (DRY 원칙)
- [ ] 변수/함수명이 의도를 명확히 표현하는가?
- [ ] 매직 넘버/문자열을 상수로 분리했는가?

### 3. 안전성
- [ ] null/undefined 처리가 적절한가?
- [ ] 예외 처리가 적절한가? (너무 광범위한 catch 없는가?)
- [ ] 시크릿/패스워드가 하드코딩되지 않았는가?
- [ ] SQL/Command Injection 취약점이 없는가?

### 4. 테스트
- [ ] 새 기능에 대한 테스트가 있는가?
- [ ] 엣지 케이스가 커버되는가?
- [ ] 테스트가 구현을 검증하는가? (단순 커버리지 채우기 아닌)

### 5. 문서
- [ ] 복잡한 로직에 주석이 있는가?
- [ ] API 변경 시 관련 문서가 업데이트되었는가?
- [ ] QUALITY_SCORE.md 업데이트가 필요한가?

## 리뷰 출력 형식

```
## 코드 리뷰 결과

### 필수 수정 (MUST)
- [파일:라인] 설명 + 수정 방법

### 권장 개선 (SHOULD)
- [파일:라인] 설명

### 참고 사항 (NICE TO HAVE)
- 설명

### 승인 여부
- [ ] APPROVED
- [x] CHANGES REQUESTED
```

## 주의사항

- 자기 작업을 리뷰할 때는 독립적 시각을 유지하세요
- "동작한다"는 것과 "잘 만들어졌다"는 것을 구분하세요
- ARCHITECTURE.md와 docs/design-docs/core-beliefs.md를 참고하세요
