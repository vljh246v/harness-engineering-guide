# Playbooks — 반복 작업 레시피

고위험이거나 반복적인 작업의 단계별 절차입니다.
각 체크리스트 항목은 `agent-failures.md`의 실제 실패에서 유래합니다.

## 목록

| 플레이북 | 대상 작업 |
|----------|-----------|
| (프로젝트에 맞게 추가) | |

## 새 플레이북 작성

1. `docs/playbooks/<kebab-case-이름>.md` 파일 생성
2. 아래 템플릿 사용
3. 이 README에 행 추가

## 템플릿

```markdown
---
name: 플레이북 이름
last_verified: YYYY-MM-DD
related_failures: F-XXXX (agent-failures.md ID)
---

# 플레이북: 제목

## 언제 사용하는가

이 플레이북을 따라야 하는 조건.

## 사전 조건

- [ ] 조건 1
- [ ] 조건 2

## 절차

### 1단계: ...

- [ ] 체크 1
- [ ] 체크 2

### 2단계: ...

- [ ] 체크 1

## 흔한 실수

| 실수 | 결과 | 올바른 방법 |
|------|------|------------|
| ... | ... | ... |

## 검증

- [ ] verify-task.sh 통과
- [ ] 관련 테스트 통과
```
